// 인증/인가
세션: - 어떤 토큰으로 로그인했는지를 DB에 저장해야함
       - 토큰으로 DB를 조회해서 사용자정보를 읽어옴.  
       - 로그인 -> 세션토큰(해시값) -> 사용자정보 캐시DB 저장
       - 토큰 -> 리소스접근 -> 캐시DB의 사용자정보를 읽음
       
        -> 브라우저에도 세션 쿠키를 저장함
            -> 세션쿠키에는 사용자정보가 없어도 됨

JWT: 서버에 클라이언트 상태를 저장안 함
        -> 브라우저에만 토큰을 쿠키로 저장함
            -> 토큰쿠키에는 사용자 정보가 들어가 있음
